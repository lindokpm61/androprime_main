### F2 — Comparison tables still overflow at 375 / 390 (P1)

c84c619 reduced `min-width` on the three named comparison tables, but the new floors are still wider than 375 (and in one case 390). Visible breakage:

| Route | File:line | `min-w` set | Overflow at 375 |
|---|---|---|---|
| `/kits` (hub) | [app/(marketing)/kits/page.tsx:419](andro-prime/09_website-app/frontend/app/(marketing)/kits/page.tsx#L419) | `min-w-[480px]` | +209px |
| `/kits/hormone-recovery` | [app/(marketing)/kits/hormone-recovery/page.tsx:583](andro-prime/09_website-app/frontend/app/(marketing)/kits/hormone-recovery/page.tsx#L583) | `min-w-[560px]` | +0px at 375 (passes — wide screen reflow OK), +12px at 768 |
| `/lp/hormone-recovery` | [app/lp/hormone-recovery/page.tsx:602](andro-prime/09_website-app/frontend/app/lp/hormone-recovery/page.tsx#L602) | `min-w-[800px]` | +449px (largest table overflow in the audit) |
| `/faq` biomarker reference table | (find — `<table.w-full.text-left.font-serif>` with thead `Marker / What it measures / Why it matters / Included in`) | implicit (no min-w) | +178px |

Three options, in increasing order of UX cost and decreasing order of effort:

- **(a) Wrap each table in `<div class="overflow-x-auto">`.** Lowest effort, standard responsive-table pattern. Tables become horizontally scrollable inside the page rather than breaking the page. **Recommended.** Pair with a `text-xs text-gray-500 lg:hidden` "← swipe to compare →" caption.
- (b) Drop `min-width` to a number that actually fits 320–375 (≈`min-w-0`). Tables become cramped on mobile but page reflows.
- (c) Mobile-only stacked-card alternative. Best UX, most code. Reserve for the `/lp/hormone-recovery` table (worst offender, paid-traffic LP).

Recommendation: **(a)** for `/kits` hub, `/faq` reference table, and `/kits/hormone-recovery`; **(c)** for `/lp/hormone-recovery` since paid traffic lands there. Decision belongs to Keith.

### F3 — Terms page: 9 sticky sidebars rendering at desktop width on mobile (P1)

The other 9 do not, so they render as 474px-wide blocks at 375 viewport.

Lines to fix (add `hidden lg:block` to each): **135, 158, 200, 288, 338, 375, 435, 484, 525**. Pattern is already proven on line 575.

`/terms` is rendered by the React `terms/page.tsx` that injects this HTML via `dangerouslySetInnerHTML` — the fix has to land in the canonical-site HTML, not the React page.
