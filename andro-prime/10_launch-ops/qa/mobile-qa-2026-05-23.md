### F2 ‚Äî Comparison tables still overflow at 375 / 390 (P1) ‚Äî **SHIPPED 2026-05-23 (commit `8b43427`) for the 3 canonical-marketing tables; `/lp/hormone-recovery` deferred**

**Investigation update.** All three tables were already wrapped in `overflow-x-auto`. The real causes were two layered band-aids defeating it:
- `app/layout.tsx:68` body has `overflow-x-hidden` ‚Äî masks any leak instead of letting wrappers scroll. Left in place as a defensive guard.
- `app/(marketing)/kits/hormone-recovery/page.tsx:569` section also had `overflow-x-hidden`, clipping the table at the section edge before the wrapper could create scroll. **Net effect: mobile users could not see the Kit 3 "Most Complete" column at all** ‚Äî a silent UX regression that the old screenshots masked. Removed this section-level hidden in commit `8b43427`.

**Fix applied:** added `max-w-full` to each of the 3 wrappers (defensive cap so they can never exceed parent width), removed the section-level `overflow-x-hidden` on `kits/hormone-recovery`, and added a `font-mono uppercase text-xs md:hidden` "Scroll to see all kits ‚Üí" caption above each table so users know to swipe. Also added `min-w-[560px]` to the FAQ biomarker table (it previously had no min-width and compressed into an unreadable narrow column at mobile widths).

**Still outstanding:**

c84c619 reduced `min-width` on the three named comparison tables, but the new floors are still wider than 375 (and in one case 390). Visible breakage:

| Route | File:line | `min-w` set | Overflow at 375 | Status |
|---|---|---|---|---|
| `/kits` (hub) | [app/(marketing)/kits/page.tsx:419](andro-prime/09_website-app/frontend/app/(marketing)/kits/page.tsx#L419) | `min-w-[480px]` | +209px | FIXED `8b43427` |
| `/kits/hormone-recovery` | [app/(marketing)/kits/hormone-recovery/page.tsx:583](andro-prime/09_website-app/frontend/app/(marketing)/kits/hormone-recovery/page.tsx#L583) | `min-w-[560px]` | section had `overflow-x-hidden` clipping the table silently | FIXED `8b43427` ‚Äî section hidden removed, Kit 3 column now reachable on mobile |
| `/lp/hormone-recovery` | [app/lp/hormone-recovery/page.tsx:602](andro-prime/09_website-app/frontend/app/lp/hormone-recovery/page.tsx#L602) | `min-w-[800px]` | +449px (largest table overflow in the audit) | **DEFERRED** ‚Äî paid-traffic LP, separate design call (stacked-card alternative may be better) |

Three options, in increasing order of UX cost and decreasing order of effort:

- **(a) Wrap each table in `<div class="overflow-x-auto">`.** Lowest effort, standard responsive-table pattern. Tables become horizontally scrollable inside the page rather than breaking the page. **Recommended.** Pair with a `text-xs text-gray-500 lg:hidden` "‚Üê swipe to compare ‚Üí" caption.
- (b) Drop `min-width` to a number that actually fits 320‚Äì375 (‚âà`min-w-0`). Tables become cramped on mobile but page reflows.
- (c) Mobile-only stacked-card alternative. Best UX, most code. Reserve for the `/lp/hormone-recovery` table (worst offender, paid-traffic LP).

Recommendation: **(a)** for `/kits` hub, `/faq` reference table, and `/kits/hormone-recovery`; **(c)** for `/lp/hormone-recovery` since paid traffic lands there. Decision belongs to Keith.
