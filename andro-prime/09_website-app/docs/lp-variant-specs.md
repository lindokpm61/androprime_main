# Andro Prime — LP Variant Page Specs
## April 2026 | Phase 0 Acquisition Landing Pages

---

## Purpose

This document defines the recommended acquisition landing-page variants for Phase 0 traffic.

These pages are intended for:

- Google Ads
- affiliate traffic
- influencer traffic
- retargeting
- selected LinkedIn campaign traffic

They are not the same as the canonical site pages. Each LP variant should behave as a direct-response page with:

- one audience
- one primary promise
- one CTA target
- reduced navigation
- minimal exits

Canonical pages remain the broader trust-and-conversion pages. LP variants are the tighter funnel versions.

Related documents:

- `09_website-app/docs/lp-architecture.md`
- `10_launch-ops/implementation-checklists/lp-implementation-checklist.md`

---

## Global LP Rules

Apply these rules to all `/lp/` pages:

- One CTA target only per page
- All primary buttons should point to the same destination
- No broad top navigation
- No hamburger / mobile drawer browse menu
- Logo can remain and link to `/`
- Sticky top CTA is allowed
- Footer legal/compliance content can remain
- No blog links
- No unrelated product links
- No bundle links unless the bundle itself is the page's only goal
- Cross-links, if used at all, should be low prominence and only appear after core conversion sections

## 1. `/lp/testosterone/`

### Audience

- ICP 1: Symptomatic Achiever
- Men who feel "not themselves," have been dismissed by their GP, and are looking for a first step

### Primary goal

- Sell Kit 1

### Hero copy direction

The hero should do three things immediately:

- mirror the emotional problem in plain language
- challenge the "normal means fine" assumption
- make the kit feel like the next obvious action

Hero should feel:

- personal
- specific
- anti-corporate
- not overly technical

Good hero direction:

- "Your GP said normal. That's not the same as good."
- "Find out exactly where your testosterone actually sits."

### CTA labels

- Primary CTA: `Order the Testosterone Health Check → £29`
- Sticky nav CTA: `Order Kit → £29`
- Final CTA section: `Order the Testosterone Health Check → £29`

### Sections to keep

- Hero
- Trust indicator / accreditation strip
- Problem section
- How it works / process section
- What the kit tests
- Results preview / dashboard section
- FAQ
- Final CTA

### Sections to cut or reduce

- Broad top navigation
- Blog links
- Founding Member links in header or hero
- Large multi-product browse sections
- Mid-page cross-sell modules
- Strong Kit 3 upsell inside the main conversion path

### Allowed internal links

Allowed:

- one low-prominence text link near FAQ or footer to `/test-selector/` once built
- one subtle low-page text link to the canonical kit page if operationally needed

Not allowed:

- links to supplements
- links to blog
- prominent links to other kits in the hero or order zone

## Operational Rule (All Five LP Variants)

- The sticky nav CTA, hero CTA, mid-page CTA if used, and final CTA should all point to the same destination
- If there is a secondary link at all, it should be text-level, visually quieter, and placed after the main persuasion sequence
- LP variants should remove any elements that encourage category browsing instead of the intended conversion

---

## Summary

These five LP variants should act as:

- narrower
- more direct
- less browse-heavy
- more campaign-specific

than the canonical site pages.

They should keep the trust and credibility needed for a health product, but remove the extra choice architecture that weakens conversion in paid and affiliate traffic.
