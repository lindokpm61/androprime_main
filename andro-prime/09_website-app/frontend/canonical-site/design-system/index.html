Borders & Spacing

Border Hierarchy

Visual weight is carried by border thickness, not shadows or color. The heavier the border, the more prominent the element.

2px — Standard
Cards (.glass-panel), inputs, standard content containers, nav border-b (static)
--border-standard: 2px solid #000000
4px — Featured
Primary CTA buttons (.btn-primary), section dividers, nav border-b (scrolled), hero section borders
--border-featured: 4px solid #000000  /  --border-section: 4px solid #000000
6px — Pull Quote
Left-border accent on .pull-quote components only. Maximum visual weight reserved for editorial emphasis.
Pull quote border-left: 6px solid #000000

Fade-up Entrance — .fade-up

This block uses .fade-up on load

Applied to hero content containers to create a smooth entrance on page load. The animation runs once — opacity 0 → 1, translateY 20px → 0 — over 0.8s with the standard easing curve.

.fade-up specs
animation: fadeUp 0.8s var(--easing-standard) forwards
opacity: 0 (initial)
transform: translateY(20px) (initial)
to: opacity 1 + translateY(0)
easing: cubic-bezier(0.16, 1, 0.3, 1)
05

Animation Tokens

Transition Durations

--duration-fast
150ms
Hover states on interactive elements that need near-instant feedback (e.g. icon buttons, toggles)
--duration-base
200ms
Default transition duration. Used on all .btn color transitions, nav state changes
--duration-slow
300ms
Panels, drawers, overlays, and anything where a slightly longer reveal reads better
Standard easing
--easing-standard: cubic-bezier(0.16, 1, 0.3, 1)
Fast out, slow settle. Applied to .fade-up and any motion that should feel purposeful rather than mechanical.

Keyframes

Used by: .fade-up — hero content entrance. Runs once on page load, not on scroll.

Used by: .status-dot-pulse in the app dashboard. Set to animation: none on all marketing pages.
